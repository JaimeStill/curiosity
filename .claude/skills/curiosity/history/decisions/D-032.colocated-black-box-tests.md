# D-032 — Colocated black-box tests — 2026-05-13

**Scope**: project

**Context**: Conventions §9 settled `tests/` as a sibling directory
to source ("Tests live in a sibling `tests/` directory, not co-located
`_test.go` files"), drawing on the tau and herald reference codebases.
At plan exit for the ecs-entity-primitives session, a narrow question
about where the new entity tests should live within `engine/tests/`
surfaced the broader observation: Go's most idiomatic test layout is
co-located `_test.go` next to source, and the centralized-directory
framing was carrying conformance cost (every new package needs a
test-tree entry; the source-to-test mapping is indirect) without
proportionate benefit. The convention was revisited before any new
test file landed under the prior pattern.

**Decision**:

1. **Test files are colocated with the source they exercise.** A
   test file lives in the same directory as the package's source.
   For example, `engine/ecs/entity/entity.go` is tested by
   `engine/ecs/entity/entity_test.go`, not by a file under a
   separate `tests/` tree.

2. **Black-box discipline is retained.** Test files use
   `package <name>_test`, importing the package under test via its
   normal import path. Only the public API is exercised. The
   inner-tier escape hatch in conventions §9 for benchmark access
   to unexported helpers (`package <name>` style) remains available
   case-by-case but is not the default.

3. **No centralized `tests/` directory.** Existing test files that
   followed the prior centralized-directory pattern migrate to
   colocation at the moment the convention is adopted; the tree
   does not carry both patterns simultaneously.

**Reasoning**:

*Idiomatic Go layout.* The Go standard library, the broader Go
ecosystem, and Go's testing tooling are designed around colocated
`_test.go` files in the package's own directory. The sibling-`tests/`
pattern (which tau and herald adopt for their own narrative reasons)
is a deliberate divergence from the language's default; the project
gains nothing from carrying that divergence forward.

*Source-test proximity.* Reading source and reading tests for the
same package now requires zero directory traversal. A developer
opening `entity.go` sees `entity_test.go` in the same folder; the
mapping from source to test is structural, not nominal.

*Convention-crystallization sweeps the tree.* When a project-scope
convention changes, every prior artifact written under the old
convention is technical debt against the new one. This session's
adoption of colocation also migrated `engine/tests/engine_test.go`
to `engine/engine_test.go` and backfilled
`engine/lifecycle/lifecycle_test.go` (the lifecycle package had no
tests under the old pattern) in the same branch. The tree carries
one pattern at a time.

*Black-box discipline orthogonal to location.* Conventions §9's
broader testing posture (black-box style, table-driven, real
dependencies preferred) is orthogonal to the location question
and remains in force. Only the location convention changes.

**Alternatives**:

- *Stay with centralized `tests/`.* Matches the tau and herald
  reference codebases. Rejected: indirection cost (every new
  package adds a `tests/<pkg>/` entry, source-to-test mapping is
  by convention rather than structural) without commensurate
  benefit at this project's scale.
- *Nested mirror under `tests/`.* Tests at
  `engine/tests/ecs/entity/entity_test.go` mirroring the source
  tree. Considered briefly at plan exit. Rejected: scales worse
  than colocation as the source tree deepens; indirection cost
  compounds with depth.
- *White-box default (`package X`).* Live option that opens
  access to unexported helpers. Rejected as default: black-box
  preserves the API-only discipline §9 already values; the
  white-box escape hatch remains available case-by-case (benchmark
  access, inner-tier helpers) per §9's existing divergence note.

**Implications**:

- Conventions §9 revised to reflect colocation as the default.
  Reference citations to tau and herald are annotated as patterns
  this project diverges from on the location axis specifically.
- Future packages add their `_test.go` files in the package
  directory without further bookkeeping.
