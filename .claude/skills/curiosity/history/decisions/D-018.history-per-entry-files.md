# D-018 — History storage: per-entry files in `decisions/` and `resets/` directories — 2026-05-04

**Scope**: project

**Context**: `history/decisions.md` and `history/resets.md` were single append-only files. After R-005 the decisions file held 17 entries (~247 lines) and the resets file held 5 (~152 lines). The trajectory is asymptotic — entries accumulate forever, file scan becomes friction, and any single edit during a session diff-touches the whole file. The single-file shape was honest at small N but stops paying off as the project's history grows.

**Decision**: Replace the single-file shape with directories. `history/decisions/` holds one file per decision named `D-###.<slug>.md`; `history/resets/` holds one file per reset named `R-###.<slug>.md`. Numeric prefix preserves chronological scan order in `ls`; slug provides discoverability via filename. No index file — the directory listing IS the index. Append-only discipline is preserved per-file: new entry = new file, supersession via new file referencing the prior. Existing entries (D-001..D-017, R-001..R-005) migrate verbatim into the new layout in the same commit that lands this decision; R-006 lands as the first new-shape reset entry in the subsequent commit.

**Reasoning**: Per-entry files match the project's single-source-of-truth discipline at finer granularity — each entry is its own discoverable artifact, slug-searchable from the filename, and edits localize cleanly in diffs. The cost (more files in the tree) is well-paid by the navigability gain; `ls history/decisions/ | grep ecs` is more direct than scrolling a 247-line file. The shape change has no implication for what content goes into entries or how cross-references work — entries reference each other by ID (e.g., "D-016"), and the ID-to-file mapping is mechanical given the naming convention.

**Alternatives**: (1) Keep the single-file shape — rejected as the trajectory is asymptotic and the friction is already measurable. (2) Create an index file (`decisions/README.md`, `resets/README.md`) with one-line summaries — rejected as index drift; a duplicated source-of-truth gradually diverges from the canonical entries. The directory listing serves the same purpose without drift risk. (3) Date-based filenames (`2026-05-04-history-files.md`) — rejected as it loses the ID prefix that anchors cross-references; the ID is what design docs and other entries refer to, not the date. (4) Frontmatter on each file — rejected as redundant with the body; the title already encodes ID, slug, and date.
