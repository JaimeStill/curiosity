**Integrated**:

- **`engine/ecs/entity/entity.go`** — `entity.ID` (uint64 packed,
  index-high / generation-low) with `Index()` and `Generation()`
  helper methods. The unexported `newID` composer is the only path
  that constructs an `ID` from `(index, generation)`; together with
  `ID` being a defined `uint64` type with no exported constructor,
  this structurally enforces D-030 §3's invariant — "all live IDs
  trace back to an allocator that knows their generation." The
  zero `ID(0)` is intrinsically invalid because the allocator
  reserves index 0 and never produces generation 0.
- **`engine/ecs/entity/allocator.go`** — `entity.Allocator` with
  three pieces of state — generation table (`[]uint32`), FIFO
  recycle queue (`[]uint32` + read head, reset-on-drain), and
  fresh-index allocation at the high end — and four methods
  (`New`, `Allocate`, `Recycle`, `Validate`). `New` pre-installs
  the index-0 sentinel so the zero ID is invalid by construction.
  `Recycle` silently ignores stale and out-of-range input per the
  contract chosen mid-session — duplicate recycles during a single
  ApplyDeferred pass are legitimate at the world layer and require
  no error. Generation 0 is skipped on wrap to preserve D-030 §1's
  "the allocator never produces generation 0" invariant.
- **`engine/ecs/entity/entity_test.go`** — colocated per D-032,
  `package entity_test`. Five tests covering allocate (first ID
  is index 1 / generation 1; distinct indices on consecutive
  allocates), recycle (reuse with bumped generation; stale-input
  no-op preserves queue state), and validate (table-driven: live,
  zero, stale, out-of-range).
- **`engine/engine_test.go`** — migrated from
  `engine/tests/engine_test.go` per D-032's colocation default.
  Same `package engine_test`; no body changes. The empty
  `engine/tests/` directory was removed in the same commit.
- **`engine/lifecycle/lifecycle_test.go`** — backfilled
  mid-closeout per a user-flagged scope gap (the
  convention-crystallization session is the natural moment to
  sweep pre-existing untested packages). Six tests covering
  Coordinator-level synchronous behavior: `Ready` pre- and
  post-`Start`; `OnStart` hook drainage by `Start`; `Shutdown`
  context cancellation; `OnShutdown` hook drainage; `Shutdown`
  timeout error path. The scope deliberately stays inside the
  Coordinator's settled synchronous shape — the hot-start
  readiness pattern at the Engine level remains forward-looking
  per R-015's Retained list, and asserting it directly is what
  R-015 flagged as brittle.

**Workflow observations this session**:

- **Convention shift surfaced at plan exit via user redirect.** A
  pre-framed narrow location question framed a smaller change than
  the user actually wanted; the open-text correction was again
  faster than the multi-option ask. Mirrors R-015's pattern
  (module-path question) in the structural-convention domain.
  When the current observable state is ambiguous, the question's
  framing should default open rather than enumerate options that
  pre-frame around prior assumptions.
- **Inter-checkpoint contract question.** `Recycle`'s stale-input
  behavior (silent ignore vs panic vs return bool) surfaced
  between C2-A (struct + New + Allocate) and C2-B (Recycle +
  Validate). The question landed where the relevant code was
  about to land, not at plan exit. Pattern: contract questions
  for code not yet in scope can wait until that code becomes the
  next checkpoint — they don't all have to land at plan exit.
- **User-flagged scope gap during closeout.** "No lifecycle
  tests?" exposed an existing untested package. The
  convention-crystallization session is the natural moment to
  sweep pre-existing packages to consistency rather than defer.
  Pattern: convention shifts should sweep pre-existing packages
  in the same session.
- **Phasing for files over budget.** `allocator.go` phased into
  two checkpoints with a natural seam — production (struct +
  `New` + `Allocate`) vs validation/recycle (`Recycle` +
  `Validate`). Both phases stayed under the source-code.md
  budget. Phasing decisions justified by published guidance
  don't warrant re-confirmation; the anticipated-checkpoints
  plan adjusts mid-session without re-approval.
- **Closeout discussion surfaced a second project-scope policy.**
  The experiment-retention shift surfaced during step-3
  evaluation of `experiments/` state — exactly the cross-cutting
  workflow question the closeout-discussion step exists to
  surface. Project-scope workflow shifts can land in the same
  session as engine work when one R-016 reset entry can carry
  both decisions.
